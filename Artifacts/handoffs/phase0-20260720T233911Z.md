## Architecture and data changes
- **Processes:** `DownloadManager.app` (SwiftUI) + embedded `DownloadEngineAgent`
  (per-user LaunchAgent) over an authenticated, versioned NSXPC interface.
- **Modules (local SwiftPM):** Domain, Application(reserved), XPCContracts,
  Persistence, SharedSecurity, SharedObservability, EngineAgent, Presentation,
  plus the ObjC `XPCSecuritySupport` audit-token shim and `TestFaultService`.
- **Database:** GRDB `DatabasePool` (WAL, FK on, busy timeout), migration v1 with
  the Phase 0/1 foundational tables + constraints (FK, CHECK, unique). Agent is the
  sole writer. Media/torrent tables intentionally absent.
- **XPC:** `NSSecureCoding` DTOs with size/count caps; handshake precedes commands;
  audit-token + `SecRequirement` client validation (fails closed); per-connection
  idempotency cache.
- **Migrations & fixtures:** migrate-from-empty, interrupted-migration rollback,
  reopen-after-unclean-close, backup/restore — all covered.
- ADRs: `Documentation/adr/0001..0004` (build system, warnings scope, XPC identity SPI, test lanes).

## Security, privacy and licensing
- **Threat-boundary changes:** XPC peer validated by audit token (not PID); the
  single SPI touchpoint is isolated in `XPCSecuritySupport` (ADR 0003). Secrets
  only in Keychain (opaque refs in DB). Logs redact URLs→scheme+host and drop error
  `userInfo`. `SecretString` never prints its value.
- **Entitlements/secrets:** no custom entitlements required in Phase 0 (non-sandboxed
  dev build); no secrets in repo, fixtures, or logs. No signing material present.
- **Dependencies/licenses:** GRDB 7.11.1 MIT (verbatim notice in
  `THIRD_PARTY_NOTICES.md`); `DEPENDENCIES.md` lists it plus dev-only tools and the
  explicitly-absent future-phase deps. Project license GPL-3.0-or-later (`LICENSE`,
  `COPYING` = verbatim GPL-3.0).
