# ADR 0001 — Build system: XcodeGen project + local SwiftPM module package

Status: accepted (Phase 0)

## Context

The specification requires `DownloadManager.xcodeproj` with a `DownloadManager`
scheme and `Stable`/`ASan`/`TSan` test plans, driven by `xcodebuild -project`
(`08-validation-commands.md`). It also requires generated files to reproduce
byte-for-byte and not be hand-edited (`05-quality-testing-release-gates.md` §6),
seven first-party modules with strict layering, and GRDB pinned exactly.

Hand-authoring a `.pbxproj` for ~10 targets is fragile and violates the
"no hand-edited generated files" rule. Linking GRDB 7 directly into Xcode
`library.static` targets fails: its internal `GRDBSQLite` C module is not
resolved into static-library targets (`missing required module 'GRDBSQLite'`),
under both explicit and implicit module builds.

## Decision

- The `.xcodeproj` is **generated by XcodeGen** from `project.yml` (the source of
  truth) and is gitignored; `make project` regenerates it.
- First-party reusable modules (`Domain`, `Application`, `XPCContracts`,
  `Persistence`, `SharedObservability`, `SharedSecurity`, `EngineAgent`,
  `Presentation`, plus the `XPCSecuritySupport` ObjC shim) live in a **local
  SwiftPM package** (`Package.swift`, product `DownloadKit`). SwiftPM resolves
  GRDB and `GRDBSQLite` correctly for library targets.
- The Xcode project hosts only what needs a bundle/entitlements/launchd or an
  XCTest host: the `DownloadManager` app, the `DownloadEngineAgent` tool, and the
  test bundles. These depend on the package's products.
- Modules are added to `Package.swift` as their sources land; every declared
  SwiftPM target has at least one source file.

## Consequences

- `xcodebuild -project DownloadManager.xcodeproj` remains the single build/test
  entry point as the spec's commands assume.
- Warnings-as-errors and Swift 6 strict concurrency for first-party code are set
  in `Package.swift` (`swiftSettings`) for module targets and in
  `Configuration/Shared.xcconfig` for the app/agent/test targets; vendor packages
  keep their own settings (see ADR 0002).
- A clean checkout runs `make bootstrap-tools` (installs XcodeGen) then any build
  target, which regenerates the project first.
